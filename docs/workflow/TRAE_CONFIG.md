# Trae 环境配置与使用指南

> **目的**: 指导如何在 Trae IDE 中高效使用本项目的工作流规则。

## 1. 规则文件 (`.trae/rules.md`)

本项目在 `.trae/rules.md` 中定义了核心开发规则。Trae 会自动读取此文件中的上下文。

**核心规则**:
1.  **先规划，再执行**: 复杂任务必须先出方案。
2.  **架构文档实时更新**: 修改代码必须同步更新 `docs/architecture/ARCHITECTURE.md`。
3.  **快速通道**: 简单任务可跳过繁琐规划。

## 2. 模式推荐

### 🚀 自动模式 (Agent Mode)
适合：
- 复杂功能开发
- 涉及多个文件的重构
- 需要深度分析 Bug

Trae 的 Agent 模式会自动遵循 "Reflect -> Plan -> Execute" 的循环，非常适合本项目的工作流。

### 💬 问答模式 (Chat Mode)
适合：
- 快速代码查询
- 简单逻辑确认
- 文档查询

## 3. 最佳实践

1.  **启动任务时**:
    - 告诉 Trae 你要做什么。
    - 如果是复杂任务，Trae 会根据规则自动进入规划模式。
    - 如果是简单任务，可以使用 `【快速通道】` 前缀明确告知。

2.  **文档维护**:
    - Trae 有能力直接读取和更新 `docs/` 下的文档。
    - 在任务结束前，要求 Trae "检查是否需要更新文档"。

3.  **常用指令**:
    - "分析一下这个功能的实现思路" (触发规划)
    - "按照这个计划执行" (触发执行)
    - "检查架构文档是否需要更新" (触发文档检查)
