# Cursor AI 提示词模板库

> **目的**: 提供标准化的提示词模板，确保工作流一致性  
> **更新**: 发现更好的模板时请及时更新此文件

## 📋 使用说明

1. 根据任务类型选择合适的模板
2. 替换模板中的 `[占位符]` 为实际内容
3. 使用 `@文件名` 引用相关代码文件
4. 始终以【规划模式】开头

## 🎯 模板分类

- [新功能开发](#新功能开发)
- [Bug 修复](#bug-修复)
- [重构优化](#重构优化)
- [性能优化](#性能优化)
- [代码理解](#代码理解)

## ⚡ 快速通道 (Fast Track)

### 模板：快速修复/小改动

适用于不涉及架构变更、逻辑流转的简单修改（如文案、注释、单行Bug修复）。

```
【快速通道】我想修复/修改 [描述]

修改原因：
[简要说明]

修改内容：
[直接说明要改什么]

验证方式：
[说明如何验证]

请直接执行并验证。
```

## 新功能开发

### 模板 1：基础功能

```
【规划模式】我想添加 [功能名称] 功能。

需求描述：
[详细描述功能需求，包括：
- 功能目标
- 用户场景
- 预期行为]

请先：
1. 分析相关代码文件（@corePlayer.ts @ipcHandlers.ts 等）
2. 理解现有的架构模式
3. 制定详细的实现计划，包括：
   - 需要创建的新文件
   - 需要修改的现有文件
   - 数据流设计
   - UI 组件设计（如果涉及）
   - IPC 通信设计（如果涉及）
   - 状态管理设计
   - 测试方案

只提供计划，不要修改代码。等待我确认后再执行。
```

### 模板 2：复杂功能

```
【规划模式】我想添加 [功能名称] 功能，这是一个比较复杂的任务。

需求描述：
[详细描述]

涉及的功能点：
1. [功能点 1]
2. [功能点 2]
3. [功能点 3]

请先：
1. 全面分析代码库，理解现有架构
2. 将任务分解为多个阶段
3. 为每个阶段制定详细计划
4. 说明阶段之间的依赖关系
5. 提供整体实现时间线

只提供计划，不要修改代码。等待我确认后再执行。
```

## 💡 使用技巧

1. **组合使用**: 可以组合多个模板，例如先理解代码，再制定规划
2. **添加上下文**: 使用 `@文件名` 引用相关代码，提供更多上下文
3. **明确目标**: 在需求描述中明确目标，避免歧义
4. **分阶段**: 复杂任务可以分阶段规划，逐步执行
